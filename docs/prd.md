**魔方还原网页应用 - 产品需求文档 (PRD)**

**版本**：v1.0  
**日期**：2026年4月20日  
**状态**：初稿（可迭代）

### 1. 产品概述
本产品是一个**浏览器端交互式3D魔方模拟器 + 教学求解器**，专注于**层先法（Layer-by-Layer / Beginner Method / 七步法）**教学。用户可以在网页上通过鼠标拖拽或移动端手指触摸直接操作魔方，支持一键打乱、逐步还原指导、下一步提示，并可实时看到算法执行动画。

**产品名称建议**：  
- 「魔方小课堂」（CubeClass）  
- 「LayerCube - 层先法魔方导师」  
- 「RubikStep - 一步一步还原魔方」

**核心价值**：  
让零基础用户像真人拿着魔方一样练习，同时通过**层先法**逐步教学，让用户真正学会如何手动还原，而不是只看最优解。

### 2. 业务目标
- **用户目标**：帮助新手在10-30分钟内学会层先法，平均还原时间从5分钟降到2分钟以内。
- **产品目标**：
  - 提供沉浸式3D操作体验（PC + 移动端完美适配）。
  - 教学模式下严格遵循层先法7步逻辑，每步给出清晰提示 + 动画演示。
  - 成为开源/免费教学工具，吸引魔方爱好者、编程学习者、教师使用。
- **成功指标**（MVP阶段）：
  - 日活跃用户 ≥ 500（上线1个月）。
  - 用户完成一次完整层先法教学的转化率 ≥ 60%。

### 3. 目标用户
- **主要用户**：魔方新手（8-25岁学生、办公族）。
- **次要用户**：魔方教学者、编程爱好者（想 fork 项目扩展）、家长/老师（给孩子教学）。
- **用户场景**：
  - 手机上随时练习魔方。
  - 电脑上边看提示边操作。
  - 教学演示（投影到大屏）。

### 4. 核心功能需求（MVP）
#### 4.1 魔方交互模块
- **3D魔方渲染**：使用 Three.js 实现真实3D立方体，支持：
  - 自由旋转视角（OrbitControls）。
  - 拖拽单层旋转（鼠标/触摸识别面 + 拖动方向）。
  - 键盘快捷键（F/R/U/L/B/D + ' + 2）。
  - 移动端触摸手势支持（单指拖动旋转层，双指旋转视角）。
- **状态管理**：实时记录54个贴纸颜色，支持 undo/redo。
- **可视化**：高亮当前操作层、显示公式动画（带缓动）。

#### 4.2 一键打乱
- 点击“打乱”按钮，生成合法随机打乱序列（25-30步，符合WCA标准）。
- 可选择打乱强度（初级/高级）。
- 打乱后自动进入“练习模式”或“教学模式”。

#### 4.3 层先法逐步还原（核心教学功能）
严格按照**层先法7步**实现，每步独立：
1. 底面十字（White Cross）
2. 底面角块（White Corners）
3. 中层棱块（Middle Layer）
4. 顶面十字（Yellow Cross）
5. 顶面角块方向（Yellow Corners Orientation）
6. 顶面角块位置（Yellow Corners Permutation）
7. 顶面棱块位置（Yellow Edges Permutation）

**每步特性**：
- **自动检测当前步骤**：系统判断魔方处于哪一步，自动切换教学面板。
- **下一步提示**：大按钮显示当前推荐公式（如 `R U R' U'`），点击“执行”自动播放动画。
- **手动模式**：用户自己转动魔方，系统实时判断是否完成当前步骤。
- **案例判断**：每步列出所有可能图案（点状、L形、一字等），点击图案直接跳转对应公式。
- **进度条**：显示7步完成进度 + 已用步数/预计步数。

#### 4.4 辅助功能
- **重置 / 还原到已解决状态**。
- **公式库**：可展开查看所有层先法公式 + 镜像公式。
- **计时器**：可选开启还原计时。
- **分享**：生成当前魔方状态链接（URL参数）。
- **PWA支持**：可安装到桌面/手机，主屏打开像App。

#### 4.5 教学增强（后续迭代）
- 语音讲解（可选）。
- 错误提示（误操作时高亮正确块）。
- 速度模式（切换到CFOP简版）。

### 5. 非功能需求
- **性能**：加载时间 < 2秒，60fps动画（支持低端设备）。
- **兼容性**：Chrome/Firefox/Safari最新版 + 移动端iOS/Android。
- **无障碍**：支持键盘操作、屏幕阅读器。
- **数据存储**：本地存储（localStorage）保存用户偏好，不收集个人信息。
- **SEO**：单页应用 + meta标签，标题含“魔方 层先法 教程”。
- **安全**：纯前端实现，无后端（或可选Node.js后端存用户打乱记录）。

### 6. 技术架构建议
- **前端框架**：React（或 Next.js） + TypeScript。
- **3D引擎**：Three.js + @react-three/fiber（推荐）或纯Three.js。
- **魔方逻辑**：
  - 魔方状态模型（参考 `ldez/cubejs` 或自行实现 Cube 类）。
  - 层先法求解器：纯JS实现7步算法（case-by-case），非Kociemba（因为用户指定“按照层先法的原理”）。
- **动画队列**：使用GSAP或Three.js内置Tween实现平滑旋转。
- **状态管理**：Zustand 或 Redux。
- **部署**：Vercel / GitHub Pages（免费）。
- **参考开源项目**（可直接fork修改）：
  - Three.js交互魔方：`joews/rubik-js`、`irisxu02/rubik`、`foo123/Rubik3`。
  - 魔方求解器JS：`torjusti/cube-solver`（可改造成LBL）。

### 7. UI/UX 设计要点
- **布局**：
  - 左侧/中央：3D魔方（占70%屏幕）。
  - 右侧面板：教学步骤列表 + 当前公式 + “下一步”大按钮。
  - 底部：控制栏（打乱、撤销、提示、计时）。
- **移动端**：响应式，右侧面板折叠成底部抽屉。
- **视觉风格**：现代简洁、魔方配色（白/黄/红/橙/蓝/绿），暗黑模式支持。
- **动画**：每步执行时高亮对应面 + 公式文字同步高亮。

### 8. 开发计划（建议4-6周MVP）
- **Week 1**：3D魔方交互 + 拖拽旋转 + 状态管理。
- **Week 2**：一键打乱 + 键盘/触摸支持 + 公式动画引擎。
- **Week 3**：层先法求解器核心逻辑（7步检测 + 公式库）。
- **Week 4**：教学UI + 提示流程 + 测试。
- **Week 5**：PWA + 移动端优化 + 性能调优。
- **Week 6**：上线 + 文档 + 开源准备。

### 9. 风险与依赖
- **风险**：触摸拖拽识别精度（尤其是移动端多层旋转）。
- **缓解**：参考成熟Three.js示例，先实现PC版再优化移动。
- **依赖**：Three.js社区示例（已有很多现成代码可复用）。
- **后续扩展**：接入摄像头扫描实体魔方、添加CFOP模式、支持4x4等。
